# EJEMPLO · SESIÓN DE SUPERVISIÓN

## Contexto

Tras varias iteraciones,
el sistema empieza a tratar tareas normales del repo
como si todas fueran cambios sistémicos.
La fricción sube
y el arranque se vuelve excesivamente burocrático.

## Activación

Se activa `VÉRTICE` por sospecha de deriva estructural.

## Observación

- El arranque carga demasiada metaestructura para tareas simples.
- La fábrica de prompts ha dejado de usarse pese a que las tareas siguen siendo complejas.
- El maestro formula demasiado contexto de sistema y poco contexto de repo.

## Estado detectado

Rigidez formal creciente con pérdida de ergonomía operativa.

## Recomendación

1. Recuperar una ruta repo-first.
2. Reintroducir agentes de bootstrap.
3. Mantener el canon, pero desplazarlo a la ruta adecuada según tarea.
4. Conservar validación y trazabilidad.

## Cierre

La versión 2.2 híbrida se justifica como ajuste correctivo controlado.
