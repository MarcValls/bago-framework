# WORKFLOW · ANÁLISIS

## id

`workflow_analysis`

## Objetivo

Comprender un problema, un artefacto, una versión o una tensión antes de rediseñar o ejecutar.

## Cuándo usarlo

- diagnósticos,
- comparativas,
- auditorías,
- revisión de coherencia,
- lectura técnica de un repositorio o pack.

## Roles mínimos

- Orquestador
- Analista
- Validador

## Entradas

- petición,
- artefacto o sistema observado,
- restricciones,
- historia relevante si aplica.

## Fases

1. Apertura y clasificación.
2. Delimitación del objeto analizado.
3. Descomposición en piezas, dependencias, riesgos y tensiones.
4. Juicio operativo.
5. Validación del análisis.

## Salidas

- diagnóstico,
- fortalezas,
- debilidades,
- límites del análisis,
- siguiente paso recomendado.

## Incidencias típicas

- análisis demasiado narrativo,
- falta de criterio de cierre,
- confusión entre observación y propuesta.

## Cierre

Se cierra con una recomendación clara de mantener, rediseñar, ejecutar o escalar.
