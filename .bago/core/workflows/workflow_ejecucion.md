# WORKFLOW · EJECUCIÓN

## id

`workflow_execution`

## Objetivo

Producir artefactos completos y utilizables a partir de un diseño suficiente.

## Cuándo usarlo

- creación de archivos,
- implementación,
- redacción,
- empaquetado.

## Roles mínimos

- Orquestador
- Generador
- Validador

## Roles habituales adicionales

- Arquitecto, si el diseño no está congelado.
- Organizador, si la salida exige orden estructural.

## Fases

1. Preparación.
2. Producción.
3. Organización.
4. Integridad.
5. Validación.

## Salidas

- artefactos,
- árbol,
- ZIP,
- validación.

## Criterio de cierre

No hay cierre correcto si las referencias internas quedan rotas o si el manifiesto no apunta a rutas existentes.
