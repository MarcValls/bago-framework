# 00 · CEREBRO_BAGO · V1

BAGO = **Balanceado, Adaptativo, Generativo, Organizativo**.

BAGO AMTEC V1 define un sistema de trabajo por fases, roles y trazas. Su objetivo no es solo producir entregables, sino hacerlo con continuidad, control de alcance y coherencia entre iteraciones.

---

## 1. Propósito

Transformar una petición del usuario en:

- objetivo operativo,
- lectura contextual,
- estrategia de intervención,
- producción útil,
- cierre ordenado,
- y rastro verificable de lo decidido.

---

## 2. Principios de V1

### P1 · Estado vivo separado del modelo

El documento que explica el estado no es el mismo documento donde vive el estado de la sesión.

### P2 · Activación mínima suficiente

Solo se activan los roles imprescindibles para la fase actual.

### P3 · Coherencia longitudinal

Toda iteración debe poder entenderse desde el estado vivo y el historial de cambios.

### P4 · Supervisión justificada

`GUIA_VERTICE` se usa por necesidad estructural, no como reflejo automático.

### P5 · Rutas canónicas

La raíz del sistema es siempre `.bago/`.

---

## 3. Ciclo BAGO

### [B] Balanceado

- clarificar intención,
- delimitar alcance,
- hacer visibles restricciones,
- detectar ambigüedad y riesgo.

### [A] Adaptativo

- decidir enfoque,
- seleccionar roles,
- proponer secuencia,
- congelar decisiones clave.

### [G] Generativo

- producir artefactos,
- mantener compatibilidad con restricciones,
- explicitar supuestos.

### [O] Organizativo

- ordenar salida,
- empaquetar,
- resumir,
- dejar validación y siguientes pasos.

## 5. Criterio de calidad

BAGO AMTEC V1 está funcionando bien cuando:

- reduce ambigüedad real,
- no dispersa roles,
- deja estado y decisiones actualizados,
- permite retomar la sesión sin reconstruir contexto,
- y conserva trazabilidad suficiente para revisión.
