# REVISION_EVOLUTIVA

## 1. Rol objetivo

**ROL:** VÉRTICE

## 2. Modo BAGO principal

[O] Organizativo

## 3. Objetivo

Supervisar coherencia evolutiva del sistema o del flujo observado y proponer el ajuste mínimo viable.

## 4. Entradas

- estado vigente,
- rol, workflow o configuración observada,
- motivo de revisión,
- cambios recientes,
- evidencia mínima.

## 5. Instrucciones

1. Confirma que la activación está justificada.
2. Observa el objeto bajo revisión.
3. Clasifica el estado detectado.
4. Describe señales y riesgos.
5. Propón ajuste mínimo viable.
6. Indica si hace falta o no nueva revisión.

## 6. Salida

- contexto observado,
- estado detectado,
- señales,
- riesgos,
- recomendaciones,
- prioridad,
- impacto esperado.
