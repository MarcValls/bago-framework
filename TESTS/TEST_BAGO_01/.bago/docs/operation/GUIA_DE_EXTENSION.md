# Guía de extensión

## Cuándo extender

Solo si una nueva pieza:

- nombra una frontera real,
- evita desorden repetido,
- mejora trazabilidad, control o integración.

## Qué se puede extender

- especialistas,
- workflows,
- validadores ligeros,
- métricas.

## Qué no debe tocarse sin protocolo

- manifiesto,
- canon base,
- taxonomía,
- contratos,
- separación entre actual, migrado y legado preservado.
