# Security Policy

## Supported Versions

Only the latest release is actively maintained and receives security fixes.

| Version | Supported          |
| ------- | ------------------ |
| v3.x    | ✅ Yes             |
| < v3.0  | ❌ No              |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Use one of the following private channels:

1. **GitHub Security Advisory** (preferred): [Report a vulnerability](https://github.com/MarcValls/bago-framework/security/advisories/new)
2. **Email**: Contact the maintainer via GitHub profile ([@MarcValls](https://github.com/MarcValls)).

### What to include

- Description of the vulnerability and potential impact.
- Steps to reproduce or proof-of-concept (if safe to share).
- Affected versions.
- Any suggested mitigation or fix.

### Response commitment

- **72 hours** — acknowledgement of the report.
- **14 days** — initial assessment and severity classification.
- **90 days** — target for patch or coordinated disclosure.

We follow responsible disclosure and will credit reporters in the release notes unless they prefer to remain anonymous.
