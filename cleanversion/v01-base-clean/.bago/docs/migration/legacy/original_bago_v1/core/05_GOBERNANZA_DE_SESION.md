# 05 · GOBERNANZA DE SESIÓN

## Objetivo

Definir cómo se gobierna una sesión BAGO AMTEC V1 para que no dependa de memoria difusa ni de improvisación acumulada.

---

## 1. Autoridades funcionales

### Usuario

- fija objetivo, restricciones y aceptación final.

### MAESTRO_BAGO

- coordina la sesión,
- mantiene foco,
- decide qué rol entra o sale,
- vela por la actualización del estado vivo.

### GUIA_VERTICE

- no gobierna la sesión,
- solo supervisa integridad cuando la activación está justificada.

---

## 2. Objetos gobernados

Se gobiernan de forma explícita:

- objetivo actual,
- modo predominante,
- fase actual,
- roles activos,
- decisiones congeladas,
- riesgos abiertos,
- cambios estructurales aceptados.

---

## 3. Reglas de sesión

1. No puede haber más de **tres roles activos** a la vez salvo justificación fuerte.
2. Todo cambio estructural aceptado debe reflejarse en el estado vivo.
3. Toda revisión evolutiva debe dejar rastro de entrada y de salida.
4. Si una tarea cambia de fase, el estado debe registrarlo.
5. Si hay conflicto entre contenido generado y estado vivo, se corrige uno de los dos de forma explícita.

---

## 4. Criterios de escalado

Escalar a revisión evolutiva cuando:

- exista deriva repetida,
- varios roles invadan fronteras ajenas,
- el sistema cambie de versión o de arquitectura,
- se pierda trazabilidad de decisiones,
- la salida se vuelva inconsistente entre iteraciones.

---

## 5. Cierre de bloque

Todo bloque relevante de trabajo debe cerrar con:

- qué cambió,
- qué quedó igual,
- qué riesgo sigue abierto,
- cuál es el siguiente paso.
