# CONTRATO DE ROL · BAGO AMTEC línea canónica previa CORREGIDO

## Objeto

Definir la forma mínima obligatoria de un rol canónico o especialista y hacerla compatible con validación documental y, si se desea, con validación semiestructurada.

## Campos obligatorios

- identidad
- familia
- versión
- propósito
- alcance
- límites
- entradas
- salidas
- activación
- no activación
- dependencias
- criterio de éxito

## Campo recomendado adicional

- relación con legado o con historia, si el rol toca migración o preservación.

## Regla de frontera

Un rol existe porque resuelve una frontera funcional. No se justifica por tono, estilo o “sensación de cobertura”.

## Regla de diferenciación

Dos roles de la misma familia deben poder diferenciarse no solo por propósito, sino por:

- entradas,
- salidas,
- criterio de activación,
- criterio de no activación.

## Regla de densidad

No se admiten roles definidos por una frase y una lista genérica si el archivo se declara operativo dentro del pack.
