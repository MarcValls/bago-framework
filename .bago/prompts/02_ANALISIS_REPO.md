# ANALISIS_REPO

## 1. Rol objetivo

**ROL:** ANALISTA

## 2. Modo BAGO principal

[B] Balanceado

## 3. Objetivo

Analizar el estado actual del repositorio y detectar huecos, tensiones, riesgos y dependencias relevantes.

## 4. Entradas

- árbol del repo,
- carpetas principales,
- configuración,
- documentación,
- contexto preparado por `ADAPTADOR_PROYECTO`.

## 5. Instrucciones

1. Resume el estado real del proyecto.
2. Detecta incoherencias y dependencias ocultas.
3. Señala riesgos técnicos o documentales.
4. Explicita huecos de información.
5. Recomienda qué debería hacer el arquitecto o el maestro a continuación.

## 6. Salida

- resumen del estado actual,
- huecos,
- incoherencias,
- riesgos,
- dependencias,
- recomendación siguiente.
