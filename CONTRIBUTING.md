# Contributing to BAGO

## Adding a new tool

1. Create `tool_name.py` in `.bago/tools/`
2. Register it: `python3 .bago/tools/auto_register.py tool_name.py`
3. Regenerate the manifest: `python3 .bago/tools/auto_register.py --gen-manifest`

---

## Tool self-test contract

Every tool **must** implement `--test` with real assertions (not just a smoke check).

### `--test` — functional test (required)

`--test` must:
1. Execute at least one real operation (use a `tempfile`-based fixture, never touch production state)
2. Verify post-conditions (file created, value returned, state changed as expected)
3. Exit `0` on success, `1` on assertion failure
4. Print a structured JSON result as the **last line** of stdout:

```json
{"tool": "my_tool", "status": "ok", "checks": [
  {"name": "creates_output_file", "passed": true},
  {"name": "output_is_valid_json", "passed": true}
]}
```

If any check fails, set `"status": "fail"` and exit `1`.  
**Never** use magic words like `"pasaron"` or `"OK"` as the sole success indicator.

### `--smoke` — quick startup check (optional)

`--smoke` (or `--check`) validates that the tool can be imported and its dependencies exist. It may exit `0` after verifying the environment without performing real operations. This is **not** a substitute for `--test` in CI.

### Template

```python
if __name__ == "__main__":
    import sys, json, tempfile
    if "--test" in sys.argv:
        checks = []
        with tempfile.TemporaryDirectory() as td:
            # 1. Real operation
            result = my_function(output_dir=td)
            # 2. Assert post-conditions
            checks.append({"name": "result_not_empty", "passed": bool(result)})
            out_file = Path(td) / "expected_output.json"
            checks.append({"name": "output_file_created", "passed": out_file.exists()})
        all_passed = all(c["passed"] for c in checks)
        print(json.dumps({
            "tool": "my_tool",
            "status": "ok" if all_passed else "fail",
            "checks": checks,
        }))
        sys.exit(0 if all_passed else 1)
```

### Why this matters

The CI integration gate (`tool_guardian.py`, `auto_register.py`, `integration_tests.py`) requires:
- `--test` exits `0`  
- Last stdout line is valid JSON with `"status": "ok"` and a non-empty `"checks"` list  
- All checks have `"passed": true`

A tool whose `--test` prints `"pasaron"` and exits `0` will be **rejected** by the integration gate.

## CI

Generated by `python3 .bago/tools/ci_generator.py --provider github`.

CI enforces:
- No `✅` or `OK` output inside exception handlers
- `tools.manifest.json` matches `.bago/tools/*.py` on disk (drift → build red)
- No duplicate keys in `COMMANDS` dict in the `bago` script
- All contracts are `FULFILLED` or `ACTIVE` — not `BREACHING` or `VIOLATED`

---

## Code style

- Python 3.9+, no external dependencies for core tools
- Tools are self-contained single-file scripts
- All tools must have a module-level docstring
