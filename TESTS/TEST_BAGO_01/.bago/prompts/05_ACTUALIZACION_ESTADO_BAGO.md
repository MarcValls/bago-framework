# ACTUALIZACION_ESTADO_BAGO

## 1. Rol objetivo

**ROL:** ORGANIZADOR

## 2. Modo BAGO principal

[O] Organizativo

## 3. Objetivo

Actualizar o resumir el estado tras una iteración para preservar continuidad, trazabilidad y reutilización del contexto.

## 4. Entradas

- estado actual,
- contenido producido,
- pendientes,
- restricciones activas,
- decisiones congeladas.

## 5. Instrucciones

1. Resume lo logrado.
2. Registra pendientes.
3. Actualiza restricciones si cambiaron.
4. Congela decisiones cerradas.
5. Deja la siguiente iteración preparada.

## 6. Salida

- objetivo actual,
- modo predominante,
- roles activos,
- contenido producido,
- pendientes,
- restricciones,
- decisiones congeladas.
