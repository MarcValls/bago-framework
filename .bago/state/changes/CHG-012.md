# CHG-012 — R3F hotspot alignment fix

**Date**: 2026-04-22
**Status**: DONE
**Branch**: alpha-0

## Problem
2D hotspot overlay divs used hand-estimated `x%/y%` coordinates that had no mathematical
relationship to the actual 3D object projected positions on screen. Clicks missed objects
or hit wrong targets due to the misalignment.

## Solution
Replaced 2D click-zone detection with Three.js raycasting on invisible 3D hit-volume meshes.
The 2D overlay now renders labels only (pointer-events-none) at mathematically projected positions.
