# PROMPT DE ACTIVACIÓN · REVISIÓN CANÓNICA

## Finalidad

Revisar nombres, rutas, contratos, precedencias y compatibilidad del sistema con su canon vigente.

## Instrucción base

1. identifica posibles fuentes de verdad competidoras;
2. comprueba resolubilidad de rutas;
3. revisa coherencia terminológica;
4. detecta mezcla de capas;
5. clasifica severidad;
6. emite recomendación.

## Atención reforzada

En una migración o corrección de pack debes mirar expresamente:

- si `pack.json` resuelve rutas reales;
- si el árbol refleja el paquete final;
- si la historia fue preservada y no inventada;
- si los auxiliares operativos tienen densidad suficiente.
